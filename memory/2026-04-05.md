## AI Company Hub Dashboard - Major Development Session (cont. from 04-04)

### Project
- GitHub repo: https://github.com/rPthrqns/ai-company-hub
- Server: Python http.server on port 3000 (`ai-company/dashboard/server.py`)
- Frontend: Single HTML file (`ai-company/dashboard/index.html`)
- Data: JSON files in `ai-company/data/`
- Commits pushed throughout session, latest: `ff93782`

### Key Features Implemented
1. **Company creation** with auto CEO/CMO/CTO agents
2. **Agent creation** via UI modal, chat natural language, and auto-detect from agent responses
3. **@mention chaining** - agents can mention each other, triggering chain responses
4. **Multilingual** - KO/EN/JA/ZH for welcome messages and UI
5. **Toast notifications** with loading/complete states
6. **Agent status system**: registering → working → active
7. **Agent session auto-activation** on creation via background `openclaw agent` call
8. **Server auto-posts agent responses** (no longer relies on agent running curl)
9. **JSON data protection**: validation, auto-backup (.bak), auto-restore on corruption

### Critical Bugs Fixed
- `openclaw agents delete` needs `--force` (not `--non-interactive`)
- `openclaw agents add` needs `--non-interactive` (not `--force`)
- PROCESSORS lock was per-company, changed to per-agent (`cid:agent_id`) to allow parallel chaining
- `register_agent` was missing `--workspace` parameter in background thread
- `createCompany()` JS had missing `try{` and `fetch` call
- companies.json corruption from unescaped agent responses in JSON
